## Curls manuales: FAIL

### Estado del servidor

API corriendo en `http://localhost:3000` (servidor dev pre-existente en la sesión).

### Causa del fallo

Los tres endpoints HTTP de US-028 **no están registrados en el controlador NestJS**. El fichero `apps/api/src/facturacion/interface/factura.controller.ts` (305 líneas) no tiene los métodos:

- `POST /api/reservas/{id}/facturas/liquidacion/aprobar-enviar`
- `POST /api/reservas/{id}/facturas/fianza/enviar`
- `POST /api/reservas/{id}/facturas/liquidacion/reenviar`

El Swagger del servidor en ejecución (`GET /api/docs-json`) sólo expone 5 rutas bajo `Facturacion` (las de US-022 y US-027), ninguna de US-028.

### Pruebas ejecutadas

```bash
TOKEN="<bearer>"

# a) Happy path: aprobar y enviar
curl -s -X POST "http://localhost:3000/api/reservas/RES_ID/facturas/liquidacion/aprobar-enviar" \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" -d '{}'
# Respuesta: 404 {"message":"Cannot POST /api/reservas/.../facturas/liquidacion/aprobar-enviar"} — FAIL

# b) Reenvío
curl -s -X POST "http://localhost:3000/api/reservas/RES_ID/facturas/liquidacion/reenviar" \
  -H "Authorization: Bearer $TOKEN"
# Respuesta: 404 {"message":"Cannot POST /api/reservas/.../facturas/liquidacion/reenviar"} — FAIL

# c) Envío separado fianza
curl -s -X POST "http://localhost:3000/api/reservas/RES_ID/facturas/fianza/enviar" \
  -H "Authorization: Bearer $TOKEN"
# Respuesta: 404 {"message":"Cannot POST /api/reservas/.../facturas/fianza/enviar"} — FAIL

# d) Sin autenticación — se espera 401
curl -s -X POST "http://localhost:3000/api/reservas/RES_ID/facturas/liquidacion/aprobar-enviar"
# Respuesta: 404 (la ruta ni siquiera existe; antes del JwtGuard) — INCONCLUSO
```

El NestJS devuelve `404 Cannot POST` (no encontrado en el router) en lugar de `401 Unauthorized`, lo que confirma que las rutas no están registradas en absoluto.

### Restauración BD

No hubo mutación (las rutas no existen, no se ejecutó ningún use case).


### 2. Crear reserva de prueba

```bash
JWT="<token del paso anterior>"
curl -s -X POST http://localhost:3000/reservas \
  -H "Authorization: Bearer $JWT" \
  -H "Content-Type: application/json" \
  -d '{
    "clienteId": "<uuid-cliente-piloto>",
    "canalEntrada": "web",
    "tipoEvento": "boda",
    "fechaEvento": "2029-09-15",
    "duracionHoras": "h8",
    "numAdultosNinosMayores4": 50,
    "idioma": "es"
  }' | jq '{idReserva: .idReserva, estado: .estado}'
```
Resultado esperado: `{ "idReserva": "<uuid>", "estado": "consulta" }`

### 3. Confirmar presupuesto — tenant SIN condiciones (esperar 409)

Si el tenant no tuviera `PLANTILLA_DOCUMENTO_TENANT` con secciones configuradas:

```bash
curl -s -X POST "http://localhost:3000/reservas/<reservaId>/presupuesto" \
  -H "Authorization: Bearer $JWT" \
  -H "Content-Type: application/json" \
  -d '{
    "extras": [],
    "metodoPago": "transferencia"
  }' | jq '{codigo: .codigo, mensaje: .mensaje}'
```
Resultado esperado:
```json
{ "codigo": "CONDICIONES_NO_CONFIGURADAS",
  "mensaje": "Configura las condiciones particulares del espacio para poder confirmar el presupuesto" }
```

### 4. Confirmar presupuesto — tenant CON condiciones (esperar 200)

Con el tenant piloto que tiene condiciones configuradas (Masia l'Encís, 14 secciones):

```bash
curl -s -X POST "http://localhost:3000/reservas/<reservaId>/presupuesto" \
  -H "Authorization: Bearer $JWT" \
  -H "Content-Type: application/json" \
  -d '{
    "extras": [],
    "metodoPago": "transferencia"
  }' | jq '{
    estado: .reserva.estado,
    condPartFechaEnvio: .reserva.condPartFechaEnvio,
    presupuestoEstado: .presupuesto.estado
  }'
```
Resultado esperado:
```json
{
  "estado": "pre_reserva",
  "condPartFechaEnvio": "<timestamp ISO>",
  "presupuestoEstado": "enviado"
}
```
Verificación clave: `condPartFechaEnvio` debe estar presente (no null) — indica que E2 fijó la fecha de envío de condiciones.
